# SDLC & DevSecOps Process Wiki

Welcome to the Software Development Lifecycle (SDLC) and DevSecOps Process documentation. This wiki provides comprehensive guidance for all phases of software delivery at our organization.

## Process Overview

Our SDLC follows a structured approach with integrated security practices (DevSecOps) throughout every phase:

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                              SDLC Master Process                                 │
├─────────────────────────────────────────────────────────────────────────────────┤
│  Product Feedback    │  Architecture   │  Develop  │  Secure, Govern  │  Test   │
│  & Discovery         │  & Design       │           │  & Validate      │         │
│  Planning            │                 │           │                  │         │
├─────────────────────────────────────────────────────────────────────────────────┤
│                          Deploy & Operate                                        │
└─────────────────────────────────────────────────────────────────────────────────┘
```

## Process Phases

### 1. [Product Feedback & Discovery and Planning](01-Product-Feedback-Discovery-Planning/README.md)
- Steps 1-9: From product feedback through sprint planning
- Key activities: Feature requests, backlog management, sprint ceremonies

### 2. [Architecture & Design](02-Architecture-Design/README.md)
- Steps 10-18: Technical design and architecture decisions
- Key activities: Technical analysis, design reviews, AI/ML considerations

### 3. [Develop](03-Develop/README.md)
- Steps 19-31: Implementation and code development
- Key activities: Coding standards, source control, feature toggles, AI-assisted development

### 4. [Secure, Govern & Validate](04-Secure-Govern-Validate/README.md)
- Steps 32-40: Security scanning, compliance, and governance
- Key activities: SAST, SCA, secrets detection, license compliance

### 5. [Test](05-Test/README.md)
- Steps 41-51: Quality assurance and testing
- Key activities: Unit testing, integration testing, performance testing

### 6. [Deploy & Operate](06-Deploy-Operate/README.md)
- Steps 52-57: Deployment and operational activities
- Key activities: Release management, monitoring, incident response
